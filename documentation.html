Color schemes

Neumorphism is generally a monochrome design, however that one color can be varied. The project currently has 7 themes including the default, and you can also make your own ones if you like. Here are the 6 custom ones:

  • Dark
  • DarkGrey
  • Peach
  • Red
  • SkyBlue
  • Violet

To use them, simply import one of them as a regular CSS stylesheet. The path [path-to-library]/Neumorphism/Themes/[theme-name].css

Making your own color schemes

Apart from using pre-defined themes, you can create your own. They are all defined by the variables in the :root element. You will need to create a separate stylesheet where you set those variables and import it with the rest of your CSS. Here are the variables you can currently set:

  • --bg - the value of your main color
  • --bg-from, --bg-to - if you want a gradient to your bg, set these variables to slightly different values. It will overwrite the --bg variable
  • --bg-tinted - a slightly darkened background color for certain elements
  • --shadow - the value of the shadows of elements. Set darker for darker bg. For lighter bg, use the transparency channel to reduce contrast.
  • --highlight - the value of the highlights of elements. Opposite the shadows. Same applies with transparency.
  • --access-main - the accessibility outline for accessibility mode
  • --access - The glow around the accessibility frame


outset-box

This is the main element of the design. It creates an outset rectangle with rounded edges. If the accessibility values are not set to 0, it also has a glowing outline. Any other element, including all of those defined in this style, can be placed inside of the container.

h2 (outset-box)

To create a glowing header for the outset box, use h2 tags.

h3 (outset-box)

Whenerver you need to number a few outset boxes, you can wrap the number in an h3 tag, which will create a circular image around it, as well as highlight the number itself. It is conflicting with h2, so you should choose one header for each box. 

<div class="outset-box">
    <h2>Example box</h2>
    <p>
        Lorem Ipsum or something,
        idk I'm not a specialist.
    </p>
    <div class="neu-button-wrapper">
        <input type="checkbox" class="neu-button">
        <i class="fa-solid fa-thumbs-up"></i>
    </div>
</div>

inset-box

An opposite of the outset box. On this website, the most common use is in forms as an input field. However, that is not the only possible use - it can be used as widely as its counterpart. When the inset box has focus, such as a text area being interacted with, it creates a slight glow effect to highlight the chosen box. 

<form class="outset-box">
    <h2>Example form</h2>
    <input name="name" type="name" class="inset-box"
        placeholder="Enter your name here.">
    <textarea name="main" rows="2" class="inset-box" id="feedback"
        placeholder="Please share your feedback"></textarea>
</form>

outset-line

This is a thick divider line that is used to separate sections of the page. You can see it all over this page. It is horizontal by dedault, but it can be vertical in a horizontal layout. The latter is usually done with flexbox. 

<hr class="outset-line">


neu-image-display

Whenever you need to incorporate an image, this display creates an outset element that the image is placed into. It should be noted that the width and height of the whole ordeal should be set in the img component, not the display.

image-attribution

If you need to site the image origin and you don't want to add text around the image, use image-attribution paragraph within the image container, and add the text within the paragraph. It will create a blur effect showing the text when you hover over the image. 

<div class="neu-image-display">
    <img src="Images/img.jpg" alt="Image description">
    <p class="image-attribution">
        <a href="[source-link]">Image by Author</a>
    </p>
</div>

neu-image-borderless

This creates the same image display, except there is no inset effect for the image. Useful for displaying images with transparency. (For example how it was made on the appeal page)

neu-button

This is a general round button with a glowing icon. Can be used on a button or on a checkbox. Always has a 50px by 50px size.
To set the glow color of the icon, set --text-glow and --text-shadow properties for the main and glow colors respectively. NOTE: The properties have to be set to the icon, not the button itself.

neu-button-wrapper

Because checkboxes can't contain children, this class has to go around the checkbox and the icon so that they can be styled together. 

<div class="neu-button-wrapper">
  <input type="checkbox"
    class="neu-button" id="btn-1">
  <i class="fa-solid fa-heart"></i>
</div>

<style>
  #btn-1 + i {
    --text-glow:   rgb(255, 50, 50);
    --text-shadow: rgb(255, 50, 50);
  }
</style>


hover-button

This is a "floating" button button that appears to be above the page. Good for sticky buttons such as "go to top".
Same as with the regular button, to set the glow color of the icon, set --text-glow and --text-shadow properties for the main and glow colors respectively. UNLIKE the regular button, the properties have to be set for the button itself. 

<button class="hover-button" id="hover-btn-1">
    <i class="fa-solid fa-question"></i>
</button>

<style>
  #hover-btn-1 {
    --text-glow:   blue;
    --text-shadow: aqua;
  }
</style>

sticky-header

This header is used as a container for whatever sticky elements you need to stick in your website. It has no appearance, and it occupies 100% of viewport width and height. To avoid blocking interactible elements on the page, the pointer events for it are turned off, so, if you are adding your own elements to it, make sure to reset that property to "all" for each element.

dropdown-nav

This class allows you to create the nav of the same type present on the page. Please note: the nav is drop-down, so it is hidden by default, with exactly 100px of it showing. If you wish to reset that, alter the "translate" property manually.

h2 (dropdown-nav)

Mark the title with h2 tags in the nav to have it appear as a display. The example can be seen in the top-left corner of the current page.

h3 (dropdown-nav)

A simple centered header for topic columns if you have a lot and you want to categorise them.

            <a href="issues.html"

p-anchor

This should be an empty divb placed right before a paragraph you want to link to. By default, anchor links move to the paragraph being at the top of the page, which is siometimes obscured - in this case, by the nav bar. This anchor is offset by 200px, placing the following paragraph roughly in the middle of the viewport. 

<div class="p-anchor" id="example-anchor"></div>
<p>
    Some text
</p>

Flexbox subclasses

The vast majority of this project, as well as most of modern web deign, relies on flexbox layout to create a responsive yet versatile design. At the beginning of the project, it was obvious that including style parameters into every second div would create a huge mess, so a few flexbox helper classes were added.

flex-horiz

Creates a horizontal layout for its children. By default, it has a gap of 30px and it can wrap to multiple lines if the window size is too narrow.

flex-vert

This one is for a vertical layout. It is the default layout of a page, so this class is used less often. Most common use is inside of a flex-horiz, to create two columns of elements side-by-side.

box-flex

Any child of a flexbox parent should have this class, unless it has a static size. Box-flex controls the scaling of the element. For example, a paragraph with no size limitations will by default stretch the entire width of the screen, and even flex-horiz won't allow to put two paragraphs side-by-side.

justify-content extensions

Flexbox layout can be used to create any part of your website. However, for that it needs a few parameters, and again, writing them in the style property of every tag quickly becomes impossible. Because of that, a few flexbox subclasses were added to cover the most useful flexbox parameter - justify-content. They can be added alongside other classes of a tag. Here are all of the current versions:

  • justify-start = justify-content: flex-start
  • justify-end = justify-content: flex-end
  • justify-s-between = justify-content: space-between
  • justify-center = justify-content: center
